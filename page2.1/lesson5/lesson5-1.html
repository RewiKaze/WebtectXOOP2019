ตอนที่ 1 : คีย์เวิร์ด static

ใช้สำหรับการกำหนดคุณลักษณะ หรือเมธอดที่ใช้งานร่วมกันในทุกวัตถุ ซึ่งสามารถเรียกใช้งานได้โดยไม่จำเป็นต้องสร้างวัตถุขึ้นมาก่อน

public static int age;

แต่ทว่าเมธอดโดยทั่วไปนั้น จะเป็น Modifier แบบ non-static

เมธอดที่มี Modifier เป็นแบบ static จะสามารถถูกเรียกใช้งาน โดยใช้ชื่อคลาสได้เลยไม่จำเป็นต้องสร้างอ็อปเจ็กต์ของคลาสนั้นขึ้นมาก่อน โดยมีรูปแบบการเรียกใช้งานดังนี้

className.methodName();

  1. แอททริบิวท์แบบ static ส่วนมากจะใช้สำหรับเป็นค่าคงที่ให้ทุกคลาสเห็นและสามารถใช้งานได้ โดยส่วนใหญ่จะมีรูปแบบการประกาศ ดังนี้
    2. public static final type ATTRIBUTE_NAME;

  2. เมธอดแบบ static ไม่สามารถ overridden ได้แต่ สามารถ overloaded ได้เพื่อที่จะนำไปแก้ปัญหา static binding
  3. ถูกออกแบบมาเพื่ออยู่บนพื้นฐานในการแชร์ทรัพยากรร่วมกันระหว่างวัตถุที่สร้างมาจากคลาสเดียวกัน

Static Initializer

เป็นบล็อกในคลาสใดๆที่อยู่นอกเมธอด และมีคีย์เวิร์ด static เพื่อนิยมให้เป็นบล็อกแบบ static

 static {
  //command
 }

สามารถเรียกใช้ได้เพียงครั้งเดียวเมื่อ JVM โหลดคลาสดังกล่าวขึ้นมา โดยจะใช้ในการดีบักโปรแกรม หรือใช้ในกรณีที่ต้องการสร้างอ็อกเจ็กต์ของคลาสขึ้นโดยอัตโนมัติ

นอกจากนี้ สามารถที่จะกำหนดบล็อกแบบ static ได้มากกว่าหนึ่งบล็อก โดยการทำงานจะถูกเรียงจากบนลงล่าง

 public class TestStaticBlock {
  static int x=5;
  static {
   x += 1;
  }
  public static void main(String args[]) {
   System.out.println("x = "+x);
  }
  static {
   x /= 2;
  }
 }

จากโปรแกรมดังกล่าวจะเห็นได้ว่า จะทำการเข้าไปทำงานแต่ละบล็อกของ static ให้เสร็จก่อนที่จะไปทำงานในส่วนอื่น ลองเดากันเล่นๆ ว่าผลลัพธ์ที่ได้จะออกมาเป็นเท่าไรกัน เดี๋ยวจะแอบไปเฉลยทีหลังนะ อิอิ
